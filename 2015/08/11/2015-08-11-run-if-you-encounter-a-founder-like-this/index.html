Run if you encounter a Founder like this

2015-08-11 Diary

  6. 6. 股权问题，创始人太贪婪

近几年互联网浪潮是一波接一波，有技术的没技术的只要你有个逼格高高的idea，弄个pitch往VC那一摆就坐等收钱了。于是乎，有技术的开始闷头搞，没技术的开始到处找。然而，真的那么容易就能拿到VC的钱吗？下面笔者就以亲身经历分享一下如何被一个不懂技术空有idea的‘entrepreneur’所坑的。

2014年6月的某一天下午，我还在家里27寸的超爽大屏幕前享受着刚刚毕业带来的快感，突然一个电话，开始自我介绍，那架势真有点像速7里面Sean把Han炸死之后给Dom打的电话：“You don’t know me, you are about to…”的感觉。

具体idea由于签了NDA不便透露，总之是很高大上的可以change the world的感觉。

接下来见面团队，一个创始人，两个做技术研究，一个做设计，还有一个有工科和MBA双学位，感觉离发家致富不远了。然而，一个星期，两次开会之后，感觉梦想渐行渐远。主要感受几点：

  • 创始人非技术出身，对行业了解非常浅显
  • 创始人的idea过于庞大，有种天马行空的感觉
  • 所有人员均是创始人从LinkedIn电话或邮件‘忽悠’过来，大家之前都未接触过
  • 除了创始人一个全职投入时间，其余的都是part-time
  • 最重要的一点，创始人几乎没有任何资源，只有一个idea（尽管有一些的文档，而且据说申请了专利，但实际还没有任何东西）

不过，每位屌丝心里都有一个一夜成名的憧憬，创始人也很有热情，说话非常具有煽动性，大家都热情高涨。那么问题来了，如何组织工作？

创始人几乎把90%的时间花在了找人上！想办法扩充团队和寻找投资人。其余的工作基本上交给我们自己。

刚开始我们是一周两次开会，地点就在John Molson楼里面随便找个自习室。每次开会大概两个多小时，一般以创始人开始。每次都是各种好消息，几天又见了几个investor，而且他们都非常非常的感兴趣。但基本上没有实质性的消息。接下来大部分的时间都花在了讨论这个idea，应该需要这样的功能，应该去掉那样的步骤。会议非常的冗长，而且没有任何的组织性，一般是想到什么说什么。这种讨论给人的感觉是创始人根本就还没有一个很成型的idea，只是有一个非常粗略的点子。

这种状态持续了近两个月，当然，期间两位研究人员还是按照创始人的想法一点一点把它变成现实，但是实现这个想法是需要投入大量时间进行研究，并不像用已有的技术开发一个产品，大部分的时间是在研究出一个技术本身，所以这样工作也是进行地非常缓慢。而我作为一个开发人员，前期能帮上的实在有限，因为连具体做什么类型的应用都没有确定，网站？App？

终于，8月份的时候，创始人带来了算是鼓励人的消息，说有investor感兴趣，并且期望在11月份看到一个prototype。如果觉得可行，将会带来资金的注入。大家开始努力，终于成功的推出了一个简单的prototype。然而，由于创始人不懂技术，他所理解的prototype和investor期待的完全不一样，以至于我们的prototype只是一个用Qt编写的桌面小程序，只是用来展示我们的核心技术。而investor期待的却是一个可以基本达到上线的产品。很明显，这是一次很糟糕的presentation。创始人在见完investor之后的前后态度明显变化，见之前觉得我们做的很牛逼，见了之后觉得很傻逼。但是我也说句心里话，确实很傻逼。

我们这个prototype的用途可以说只能用来内部展示或者验证我们的核心技术的效果和可行性，对于拿去见investor，人家耐着性子听你说完算是尊重你了。Investor期待的还是能够很快上线盈利的产品。

接下来，我们吸取教训，开始认真研究做出面对消费者的产品。刚开始无非两种形式：网站或者App。然后酸开始爽了，创始人几乎一个星期变一个想法。我一开始就坚决支持专注以移动方向，也就是做手机App。而创始人的重心还是放在找技术人员上，期待能逮住一个技术大牛几个晚上能给你捣鼓出一个产品出来，所以由于受到不同技术人员的影响，创始人对于做网站还是App这个决定一直摇摆不定，这样又过去两个月。。

最终大家决定做网站，三月份的时候，终于找来了一个技术牛人，经验丰富，执行力很强（还是中国人最靠谱）。这个时候，创始人也带来了好消息，他已经召集了近20个天使投资人，打算在6月初把所有人召集在一间大会议室，对我们的网站和核心技术进行展示。

当然，高潮总是在最后，就在见天使们前一个星期，创始人不知从哪揪了个律师过来，以宣布的形式委婉地讲述了公司股权的分配。创始人一人占90%，其余所有人一起占10%，拿到投资之后，‘其余所有人’的10%也会被稀释。我偷偷掐指一算，这他妈连1%都不到了。

结局还是很”Happy”的，所有技术人员（其实加我也就三个）全部退出了，其他人情况不明。耗时一年的创业经历正式以失败告终，而我却感觉浑身轻松，得到了解放。

总结下来，如果你碰到的创始人符合下面任意一点，请你转身离开，留下风吹头发潇洒的背影。

1. 创始人完全是外行，对要做的行业没有一个清晰地认识

不懂技术没关系，但创始人一定要对这个行业有一个很清晰的认识。对市场的空白，竞争者的优劣势，用户的需求痛点等等都要有一个明确的了解。如果没有，那么今后会很难和他沟通，

2. 创始人的idea过于庞大和复杂

创业公司是小团队，各方面资源都有限，讲究的要快速开发出产品的原型，尽快和终端用户接触经行试错。如果创始人的想法过于复杂和庞大，很可能产品都没有推出大家就疲了，最初的热情就会渐渐退去。

3. 创始人对自己的idea摇摆不定

如果连创始人自己都不清楚自己要做什么，团队其他人怎么知道。而且，对idea的摇摆不定会造成时间，资源的大量流失，到头来花了很长的时间却做出一个四不像。

4. 创始人相信投资者胜过相信技术团队

这种情况很容易发生在创始人身上，因为创始人就是想如何尽快拿到钱，所以会不自觉去迎合投资者，而投资者和团队技术人员角度又不一样。投资者的期望是尽快盈利，而团队技术人员可能是跟着创始人的初心，用心去打造一个产品。如果创始人过分相信投资者的意见，势必与其团队内部的技术人员有冲突，技术人员很容易弃创始人而去，而创始人也会慢慢失去初心。

5. 创始人只带来好消息，然并卵

如果创始人每次开会只说所谓的好消息，而这些消息并非一些实质性的，听一两次觉得可以激励一下大伙，但是时间长了，大家都会“切~”。比如，创始人总是说今天又见了几个investor，又有多少人非常感兴趣，有人愿意投我们了只要我们能做出一个原型。这些话基本等于没说。创始人应该和团队分享任何有价值的消息，不管好的还是坏的。创业团队本来就小，保证交流的透明性很重要

6. 股权问题，创始人太贪婪

尽量在创业初期将股权分配好，越到后期越来协调，每个人付出不一样，一开始分配好股权有利于后期的顺利发展。创始人初期开始最好把握在50% - 70%，留给团队越多，越能培养团队的凝聚力。

写了这么多，希望大家碰到类似的情况吸取笔者的教训，有理想有抱负固然是好，但是一定要跟对人。发现情况不对，立马抽身，千万不可因一时贪念越陷越深，浪费了更多的时间和精力。
